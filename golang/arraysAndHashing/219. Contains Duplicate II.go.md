# 219. Contains Duplicate II

## thoughts

### 2層迴圈
第1層 loop 每一個元素, 第2層loop該元素後的 k-1個元素
如果找到就回傳, 如果第二層的元素超出陣列長度就 break 繼續下一個

### hashmap
用一個 hashmap 記錄每一個遍歷的元素, 記錄它的數值對應它的位置
如果再遇到同個數值, 則取出位置與當前位置比較是否相差 <= k, 是就回傳
如果相差>k, 更新元素的位置為新的當前位置(不用再考慮舊的那個元素, 因為後續如果再遇到相同的數值跟舊的元素的位置相差必定>k)


### sliding window + hashset
以sliding window的角度去思考: 如果兩個相同數值在同一個窗口, 則他們的distance必定符合條件
如果在窗口內遇到一樣數值則直接回傳
每次遍歷元素時都將其加入hashset
如果hashset的size>k, 則移除掉第i-k個數值的資料, 也就是移除掉窗口最左邊的資料, 讓窗口始終維持在k個資料的大小
